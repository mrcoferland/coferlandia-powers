# Validate MVP Workflow

Purpose: define the smallest experiment that tests demand without overbuilding.

1. State the core hypothesis.
2. Choose the smallest test: landing page, script, dashboard, bot, API, manual concierge service, spreadsheet, newsletter, or clickable demo.
3. Define what to build and what not to build.
4. Pick one target user and one acquisition channel.
5. Define one main metric.
6. Define success criteria, kill criteria, and a next step if successful.
7. Build a 48-hour plan and a 7-day plan.
8. Require human approval for spending, contracts, or sensitive operations.

Output should use `templates/experiment-plan.md`.
